# 에디의 하루 — 플래너 통합 · IA & 유저플로우 v0.1

> PRD v0.1 기반 정보구조(IA)와 핵심 유저 플로우 문서.
> 원본 컴패니언 앱 IA 표준(측정 중심)을 ADHD **계획→실행** 흐름으로 재구성함.

---

## 1. 정보구조 (사이트맵)

### 1.1 하단 탭 구조 (4탭)

```
┌────────────────────────────────────────────────────┐
│   오늘   │   캘린더   │   에디   │   더보기          │
│ (Today) │(Calendar) │(Eddie) │  (More)           │
└────────────────────────────────────────────────────┘
```

**탭을 4개로 나눈 핵심 설계 의도** — '오늘'과 '캘린더'를 분리했다. ADHD 사용자는 실행 뷰(지금 뭐 해야 해)와 계획 뷰(이번 주 조망)를 한 화면에 섞으면 과부하가 온다. '오늘'은 *실행*, '캘린더'는 *계획*으로 역할을 갈랐다.

| 탭 | 역할 | 주요 섹션 (Depth 2) |
|----|------|---------------------|
| **오늘** | 지금 실행 (실행 뷰) | 에디 시계·다음 출발 카운트다운 / 오늘 타임라인 / 할 일 3개 / 복약 카드 |
| **캘린더** | 계획·조망 (계획 뷰) | 주 뷰 / 월 뷰 / 일정 등록 / 타임블록 편집 |
| **에디** | 동행·동기부여 | 캐릭터 상태 / 격려 / 회복형 스트릭 / 저녁 회고 |
| **더보기** | 관리·설정 | 루틴 빌더 / 복약 설정 / 제자리 맵 / 데이터·프라이버시 / 알림 설정 |

⚠️ 5탭 이상 금지 — 캡처(할 일 빠른 추가)는 탭이 아니라 **모든 화면 우하단 FAB(플로팅 버튼)**로 상시 노출.

### 1.2 사이트맵 (Depth 1~3)

깊이 원칙: 핵심 실행 동작(할 일 캡처, 블록 수락, 복약 확인)은 **1탭**, 일정 등록은 **최대 3탭** 안에 끝난다.

- 오늘 → 에디 시계 / 출발 카운트다운 · 오늘 타임라인 · 할 일 우선 3개 · 복약 카드
- 캘린더 → 주 뷰 · 월 뷰 · 일정 등록/편집(타임블록 · 준비/이동시간)
- 에디 → 에디 상태·성장 · 회복형 스트릭 · 저녁 회고
- 더보기 → 루틴 빌더 · 복약 설정 · 제자리 맵 · 데이터·프라이버시 · 알림 설정
- ＋ 빠른 캡처 FAB → 상시 노출

## 2. 핵심 유저 플로우 (요약)

### Flow 1 — 첫 사용 (온보딩 · 최소 설정 · 권한)
핵심 원칙: **어려움 1~2개만 먼저** 고르게 해 초기 과부하를 막는다.
분기: 캘린더 미연동(내부 캘린더로 시작) / 알림 거부(제한 모드 + 재요청 경로 보관).

### Flow 2 — 아침 지각방지 (에디 시그니처)
핵심: 회의 시간만 넣어도 **출발 시각을 역산**하고, 빠른 시계로 시간맹을 보정한다.
- 에디 시계 표시(실제보다 5~10분 빠름) → 다음 일정 감지 · 이동+준비시간 역산 → 출발 시각 카운트다운 카드
- 여유: 가벼운 안심 톤 / 임박: 출발 넛지 → 나가기 전 체크(약·준비물 원탭) → 출발·에디 응원
- 일부 놓침: 놓친 항목만 강조 · 자책 없이 재알림

⚠️ 빠른 시계 규칙: 정확한 오프셋(몇 분 빠른지)을 **절대 노출하지 않는다**. 노출 시 사용자가 머릿속으로 역산해 무력화됨(위험 R-01).

### Flow 3 — 계획: 할 일 캡처 → 타임블록 (계획-실행 연결)
핵심 차별점: 할 일을 **자동 배치하지 않고 '제안 후 수락'**한다(통제감 보존, 위험 R-07).
- ＋ FAB 탭 → 빠른 캡처(음성/텍스트) → 1초 내 저장
- 할 일 과다 시 오늘 3개만 우선 노출(압도 방지) → 에디가 빈 시간에 블록 제안
- 수락 / 수정 / 다음날로(이월 · 자책 없음)

### Flow 4 — 실패 · 회복 (RSD 방어선 · 가장 중요)
핵심: 실패해도 **혼내지 않고** 회복 경로를 준다.
- 놓침 감지 → 자책 문구 없는 화면 → "괜찮아, 지금 할래? 옮길래?"
- 지금 15분만(미니 타이머) / 저녁으로(재배치) / 내일로(이월) / 오늘은 패스(비난 없이 접기)
- 스트릭 리셋 아님 · '회복' 표시. 복귀 환영도 담백하게(과도한 정서적 의존 유도 금지).

### Flow 5 — 저녁 회고 · 내일 미리보기
핵심: **'완벽'이 아닌 '괜찮은 하루'**(핵심 1개 완수) 기준으로 집계한다.

## 3. 타임블록 라이프사이클 (상태)

`Planned → Active → Done` / `Active → Missed → Recovering → Done` / `Missed → Deferred → Planned` / `Missed → Skipped`.
핵심은 '놓침'이 **종착역이 아니라 회복 가능 상태**라는 점.
에디 감정 상태는 블록 상태에 종속: `Done→기쁨`, `Missed→걱정 아닌 담담`, `Recovering→응원`, `Skipped→괜찮아`.
